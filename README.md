# 업비트 볼린저밴드 단타 자동매매 봇

KRW 마켓에서 볼린저밴드 + RSI + MACD + 거래량 복합 시그널을 기반으로 자동 매수/매도를 실행하는 스캘핑 봇입니다.

## 작동 원리

5초 간격으로 아래 파이프라인을 반복합니다:

```
코인 선정 → 캔들 수집 → 지표 계산 → 시그널 판단 → 리스크 확인 → 주문 실행
```

### 매수 조건 (AND - 동시 충족)

| 지표 | 조건 |
|------|------|
| 볼린저밴드 %b | ≤ 0.0 (하단밴드 이탈) |
| RSI | ≤ 30 (과매도) |
| MACD 히스토그램 | 이전봉 대비 증가 (반등 징후) |
| 거래량 | 20봉 평균 × 1.5 이상 시 가중 |

시그널 점수 70점 이상이면 풀 사이즈, 50점 이상이면 하프 사이즈로 진입합니다.

### 매도 조건 (OR - 하나라도 충족)

- **익절**: BB 중심선(20MA) 도달 / 상단밴드 도달 / RSI ≥ 70
- **손절**: 매수가 대비 -2% / BB 하단밴드 추가 하락 돌파
- **시간 손절**: 매수 후 30분 경과 시 무조건 청산

## 리스크 관리

| 항목 | 기본값 |
|------|--------|
| 1회 매매금 | 500,000원 |
| 최대 동시 보유 | 3개 코인 |
| 일일 최대 거래 | 20회 |
| 일일 최대 손실 | -50,000원 (도달 시 당일 중단) |
| 손절 후 쿨다운 | 해당 코인 15분 매수 금지 |
| 연속 3회 손절 | 전체 매매 30분 중단 |

## 프로젝트 구조

```
├── config/
│   └── settings.py          # 전체 설정 (API키, 매매금, 손절률 등)
├── core/
│   ├── market_selector.py   # 거래량 상위 코인 자동 선정
│   ├── data_collector.py    # 5분봉 캔들 수집 + 지표 계산
│   ├── signal_engine.py     # 매수/매도 시그널 스코어링
│   ├── order_executor.py    # 시장가 주문 실행
│   └── risk_manager.py      # 포지션/손절/쿨다운/일일한도 관리
├── indicators/
│   ├── bollinger.py         # 볼린저밴드 (상/하단, %b, 밴드폭)
│   ├── rsi.py               # RSI
│   ├── macd.py              # MACD (라인, 시그널, 히스토그램)
│   └── volume.py            # 거래량 비율
├── utils/
│   ├── upbit_client.py      # 업비트 API 래퍼 (rate limit 자동 관리)
│   └── logger.py            # 콘솔 로그 + CSV 거래 기록
├── logs/
│   └── trades.csv           # 거래 내역 자동 기록
├── main.py                  # 메인 실행 파일
├── requirements.txt         # 의존성 패키지
└── .env.example             # API 키 템플릿
```

## 설치 및 실행

### 1. 의존성 설치

```bash
pip install -r requirements.txt
```

### 2. API 키 설정

업비트 [Open API 관리](https://upbit.com/mypage/open_api_management) 페이지에서 API 키를 발급받은 뒤:

```bash
cp .env.example .env
```

`.env` 파일을 열어 발급받은 키를 입력합니다:

```
UPBIT_ACCESS_KEY=발급받은_access_key
UPBIT_SECRET_KEY=발급받은_secret_key
```

API 키 권한은 **자산조회 + 주문**을 허용해야 합니다.

### 3. 실행

```bash
python main.py
```

종료: `Ctrl+C`

## 설정 변경

`config/settings.py`에서 주요 파라미터를 조정할 수 있습니다:

```python
"buy_amount": 500_000,      # 1회 매매금 (원)
"max_positions": 3,          # 최대 동시 보유 코인 수
"stop_loss_pct": -2.0,       # 손절 기준 (%)
"time_stop_minutes": 30,     # 시간 손절 (분)
"top_n": 5,                  # 거래 대상 코인 수
"candle_interval": 5,        # 캔들 단위 (분)
"loop_interval": 5,          # 메인루프 간격 (초)
```

## 실전 투입 전 권장 순서

1. 로그만 확인하며 시그널 검증 (코드에서 주문 실행 주석 처리)
2. 최소 금액(5,000원)으로 실매매 테스트 — 3일
3. 목표 금액으로 실전 전환

## 주의사항

- 업비트 API 요청 제한: 분당 600회 (봇 내부에서 자동 관리)
- 왕복 수수료 0.1% — 최소 0.1% 이상 수익이 나야 본전
- 자동매매는 항상 손실 위험이 있으므로 감당 가능한 금액으로 운용하세요
- API 키는 절대 외부에 노출하지 마세요
